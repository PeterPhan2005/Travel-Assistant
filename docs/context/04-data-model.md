# Data Model

## Core server entities

- `users`: Firebase UID, created_at.
- `user_preferences`: budget, food preferences, travel pace, max distance.
- `trips`: destination, start/end dates, status.
- `itineraries`: owner, trip, version, created_by.
- `itinerary_items`: POI, start/end, travel time, notes generated by system.
- `pois`: canonical identity, coordinates, category, city, status.
- `poi_aliases`: Vietnamese/English/local aliases and normalized search terms.
- `poi_sources`: provider, external ID, retrieved_at, confidence.
- `poi_attributes`: air_conditioning, outdoor, spacious, family_friendly, source.
- `menu_items`: POI, dish, price, currency, source_type, updated_at.
- `narrations`: POI, language, content, verification status, generated_at.
- `narration_sources`: title, publisher, reference, accessed_at.
- `local_culture_items`: city/area, topic, content, verification status.
- `travel_packages`: city, version, manifest, published_at.
- `agent_runs`: request ID, agent name, latency, status, token usage; no raw secrets.

## Privacy

- Do not create a location-history table for MVP.
- Voice audio is not stored.
- Transcript is not persisted by default.
